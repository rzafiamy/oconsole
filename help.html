AI Agent Workflow

From Question to Answer: A Step-by-Step Visual Breakdown

Stage 1: User Input & Initial Handling

The process begins when you enter a prompt and press Enter. The application captures this input and determines if it's a task for the AI.

‚Ä∫ what time is it ?

Internal Process:

  • The main loop in manager.py receives the input string.
  • It calls handle_meta_commands to check for special commands (e.g., /help).
  • Since it's not a special command, it's treated as a task and passed to process_task.

Stage 2: The Agent Wakes Up

A new, isolated agent session is created for your task. This ensures context from previous questions doesn't interfere.

Internal Process:

  • run_agentic_mode is called.
  • A new GenericClient instance is created. Its history is fresh.
  • The initial conversation history is built for the API call:

role: system

content: "You are oconsole, an expert..."

role: user

content: "what time is it ?"

The agent makes its first API call. Based on its system prompt, it knows it must first create and explain its plan to the user before taking any action.

Internal Process:

  • The LLM analyzes the request and its rules.
  • It follows **Rule #2**: "Explain Your Plan".
  • It selects the explain_plan tool and generates the text for the plan.
  • The manager receives this instruction and displays the following output:

With the plan established, the agent makes a second API call. It now proceeds to execute the first action step of its plan.

Internal Process:

  • The LLM follows **Rule #3**: "Execute & Adapt".
  • It selects the run_safe_command tool with the argument date.
  • The CommandExecutor runs the command using Python's subprocess module.

Stage 5: The Final Answer ‚úÖ

The agent now has the information it needs. In its third and final API call, it determines the task is complete and formulates a user-friendly summary.

Internal Process:

  • The LLM follows **Rule #4**: "Final Answer Only".
  • It selects the answer_question tool.
  • It synthesizes the raw command output into a natural language response.

This workflow ensures a transparent, predictable, and robust interaction for every user request.
